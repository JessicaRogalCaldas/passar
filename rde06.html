É possível gerenciar as versões de um código ou documento realizadas por um desenvolvedor ou >equipe de desenvolvedores.

Para isso é possível realizar o Controle de Versão:

Assim, através de um software, um repositório local vai ser criado e a partir do seu arquivo, um histórico de versões.

Exemplo:

Também pode-se fazer um Repositório Remoto, onde necessita de um servidor específico para trabalhar com esse repositório.

Sincronização:

Desta forma, o software de controle de versão vai sincronizar os repositórios, fazendo com que tudo que esteja no repositório local armazene no remoto.

O software desta forma controla e permite quem irá fazer ou não as alterações, sempre notificando os usuários para não gerar problemas.

TIPOS DE SOFTWARE DE CONTROLE DE VERSÃO

  • Centralizado
  • Distribuído

CENTRALIZADO

Possui um repositório central em um servidor com controle de versões.
Nele todos os usuários se conectam e trabalham no servidor juntos. Quando o servidor para, todas as versões também vão parar, sendo totalmente dependente do servidor

Exemplo:


Software Subversion

DISTRIBUÍDO

Possui um repositório local com controle de versões.
Atualizações no repositório central após concluir seu trabalho.
Independente do servidor, caso ele caia o trabalho pode continuar sendo feito no repositório local e depois ser atualizado no central.
Exemplo:
Software Mercurial, Git.

GIT

GIT é o próprio software de versionamento.
Abreviação de sistema global de versionamento
É um software livre de versionamento que surgiu para o desenvolvimento do Kernel do Linux, foi projetado e desenvolvido por Linus Torvaldsem 2005.

Cada diretório de trabalho do Git é um repositório com um histórico completo e habilidade total de acompanhamento das revisões, não dependente de acesso a uma rede ou a um servidor central.

GITHUB

Plataforma de hospedagem de código-fonte e arquivos com controle de versão.
Surgiu em 2008 e foi adquirido em 2018 pela Microsoft por US$ 7,5 bilhões.

COMANDOS GIT

  • commit: Torna permanente um conjunto de alterações em um repositório.
  • push: Envia alterações para o servidor em um repositório remoto.
  • fetch: Verifica se o seu repositório local está em sincronia com o remoto.
  • pull: Sincroniza o repositório local com o remoto atualizando as versões.
